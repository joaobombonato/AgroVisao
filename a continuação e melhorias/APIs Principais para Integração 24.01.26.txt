APIs Principais para Integração:

AgroAPI (Embrapa)
Plataforma brasileira com APIs freemium para agronomia.
Agritec: Recomenda cultivares mais aptas para 12 culturas (inclui soja, milho, trigo, feijão). Inclui modelos de recomendação baseados em clima/solo.Época ideal de plantio por município/região (baseado no ZARC). Recomendação de adubação e correção de solo (com base em análise de solo + meta de produtividade)
API Agrofit: banco completo de defensivos agrícolas(agrotóxicos, herbicidas, fungicidas, inseticidas, bioinsumos) registrados no MAPA. Inclui princípio ativo, indicações de uso por cultura/praga, classificação toxicológica, classe ambiental, detentor do registro etc. Excelente cobertura nacional. → ideal para criar recomendações dinâmicas de proteção vegetal nos seus “guias técnicos”.
SmartSolos Expert API → Melhor para Tipos de Solo
Sistema especialista que classifica perfis de solo conforme o Sistema Brasileiro de Classificação de Solos (SiBCS).
Útil para identificar tipos de solo (Latossolo, Argissolo, Nitossolo, etc.) por região.
Outras APIs: AgroTermos (glossário técnico), BlueStar Sting (modelos específicos).
Outras úteis: ClimAPI (clima), SATVeg (índices de vegetação via satélite), Agrofit (fitossanitários), Bioinsumos (insumos orgânicos).
Outras relacionadas: API Bioinsumos, e há menção a API de Classificação de Solos da Embrapa (facilita curadoria e classificação taxonômica de solos brasileiros).→ Acesse via plataforma AgroAPI (geralmente requer cadastro/chave). É a base mais autorizada e atualizada para Brasil.
Acesso: Cadastre-se em portal.agroapi.cnptia.embrapa.br, gere token e use endpoints REST. Docs: Tutorial inicial.
Acesso: https://www.embrapa.br/agroapi (registre-se para obter documentação e chaves). Gratuito para a maioria dos usos de integração
SoilGrids API (ISRIC - World Soil Information) → https://soilgrids.org/ ou REST API (rest.isric.org).
Melhor opção global/gratuita para tipos de solo e propriedades por coordenadas/região (250m resolução). Fornece textura (argila, silte, areia), pH, capacidade de água, carbono orgânico, densidade etc. para qualquer ponto no Brasil (muito útil para regionalização por estado/microrregião). Excelente para mapas e recomendações de fertilidade.Há pacotes Python/R para facilitar (soilgrids package)

Dicas Importantes
A maioria das informações de cultivares por micro-região vem do RNC (Registro Nacional de Cultivares) do MAPA + recomendações da Embrapa. A Agritec já entrega uma boa parte filtrada.
Defensivos: Agrofit é a fonte mais completa e atualizada do Brasil.
Solo por região: O SmartSolos é bom para classificação, mas para mapas detalhados você pode precisar integrar também o Mapa de Solos do Brasil (IBGE/Embrapa) em shapefile.

Imagens de Satélite e Monitoramento de Cultivos:
Brazil Data Cube (INPE): Plataforma gratuita com cubos de dados de satélites (ex.: Landsat, Sentinel, CBERS) para classificação de uso do solo, monitoramento de LULC (Land Use and Land Cover) e agricultura sustentável. Tem APIs para análise de séries temporais, ideal para detectar mudanças em lavouras por região. Acesse via https://data.inpe.br/bdc/.

OneSoil API: Gratuita para monitoramento de campos, com dados de satélite para yields, sementes e fertilizantes. Adaptada ao Brasil

FAOSTAT (FAO)
Produção, rendimento e área plantada de soja e milho no Brasil (séries históricas anuais).
Use para benchmarks de produtividade ou calculadoras de meta de colheita.
Acesso: downloads gratuitos em CSV/ZIP direto no site (https://www.fao.org/faostat/en/#data → Production → Crops and livestock primary → Brazil → Soybeans / Maize). Há endpoints REST programáticos (fenixservices.fao.org/faostat/api/v1).

USDA NASS APIs (EUA, mas dados globais adaptáveis)
Dados estatísticos de produção para milho, soja etc.
Quick Stats API: Produção, rendimento, área plantada por estado/região (1997+).
Outros: Cropland Data Layer (mapas de cultivos), VegScape (NDVI para condições de lavoura).
Cálculos: Integre para estimar yields.
Preço: Gratuito (com chave API).
Acesso: nass.usda.gov/developer. Use em Python/R via pacotes como rnassqs
USDA Quick Stats API + FAS OpenDataWeb
Estatísticas detalhadas de colheita, rendimento e exportação de soja e milho (inclui dados do Brasil via FAS).
Bom para comparar safras brasileiras com mundiais.
Quick Stats: https://quickstats.nass.usda.gov/api
FAS: https://apps.fas.usda.gov/opendataweb/
Gratuito com chave API.

MapBiomas → Mapas anuais de uso do solo, cobertura vegetal, desmatamento, e agora MapBiomas Solo (carbono orgânico). Dados para download/API em alguns casos → Excelente para análise por bioma/região.

Cultivares de Soja e Milho por Microrregiões
Não existe uma API perfeita e gratuita ultra-específica por "microrregião IBGE" atualizada automaticamente, mas combine estas:

Embrapa AgroAPI (Agritec ou endpoints de cultivares) + Portal Cultivares Embrapa → https://www.embrapa.br/cultivares. Muitos dados de recomendação estão aqui ou via ZARC (Zoneamento Agrícola de Risco Climático - MAPA/Embrapa), que indica cultivares recomendadas por região/município com base em clima/solo.
BrAPI (Breeding API - https://brapi.org/) → Padrão internacional para bancos de germoplasma e cultivares. Algumas instituições brasileiras/Embrapa podem expor dados assim. Boa para interoperabilidade se você quiser integrar variedades de soja/milho.
Recomendação prática: Use Embrapa + IBGE API (para microrregiões/municípios) + dados ZARC (downloads ou scraping controlado). As listas oficiais de cultivares recomendadas para soja/milho saem anualmente por região (Centro-Oeste, Sul etc.).


Radar Meteorológico "Ao Vivo" (Precipitação em Tempo Real) – Opções Gratuitas

RainViewer API (melhor opção gratuita para radar ao vivo no Brasil)
Cobertura: Inclui radares brasileiros (ex.: São Paulo, Rio, Brasília, etc.), atualiza a cada 5-10 minutos.
O que entrega: Imagens de radar (PNG/Tile para mapas), dados de precipitação (intensidade, acumulado), histórico curto e previsão de movimento da chuva.
Tier gratuito: Sim, generoso para não-comercial ou baixo volume (milhares de requests/dia sem chave em alguns endpoints; plano free com limite razoável).
Documentação: https://www.rainviewer.com/api.html (tem endpoints para mapas animados e dados JSON).

Open-Meteo API (com radar/precipitação de alta resolução)
Gratuita, sem chave de API, open-source.
Tem endpoint de "radar" via integração com dados EUMETSAT/NOAA, mas mais focado em precipitação nowcast (previsão imediata) do que imagem raw de radar.
Útil como complemento: https://open-meteo.com/en/docs (procure por "precipitation" e "nowcast").
Não é radar clássico, mas dá estimativa em tempo real sem custo.

CPTEC/INPE MapSAT e Portal de Satélites (visual, mas integrável)
App MapSAT mostra radar + satélite em um mapa único, gratuito.
Não tem API oficial, mas as imagens são públicas e podem ser consumidas via URL direta (ex.: tiles ou PNG gerados dinamicamente) para embed no seu SaaS.
Alternativa: Use o portal INPE de acesso gratuito a dados de radares – baixe datasets periodicamente ou scrape ético para atualização.

Outras menções rápidas (freemium):
Meteoblue Maps API: Tem radar ao vivo, mas tier free limitado.
Zoom.Earth: Mapas interativos com radar Brasil, mas sem API aberta.

Sugestão de Stack Gratuito para Agregar Agora

Radar ao vivo principal: RainViewer API (fácil de integrar, visual impactante no dashboard do SaaS).
Satélite extra: Brazil Data Cube (INPE) – fortalece o "made in Brazil" e dá dados processados para soja/milho.
Clima complementar: Open-Meteo (zero custo, alta precisão em precipitação).
Bônus Embrapa AgroAPI: Continua gratuita (freemium com 1 mês full + limite depois), use ClimAPI e SATVeg para NDVI de satélite + zoning (não tem radar, mas agrega valor em produtividade estimada).

Como montar a calculadora no seu SaaS
Combine 2–3 APIs:
Clima/Satélite (Agromonitoring ou NASA POWER) + dados de solo/cultura (Embrapa ou FRST) + modelo matemático/ML → gera recomendação de adubação, previsão de yield, alerta de pragas etc.
Exemplo simples: NDVI baixo + precipitação acumulada baixa → recomendar irrigação + N extra.

Outras úteis:
Farmonaut ou GeoPard → APIs de agricultura de precisão/satélite.
APIs de preços de commodities → CEPEA (soja, milho) ou CME Group futures (pago).
IBGE API + BrasilAPI → Para dados geográficos, microrregiões, municípios.
Leaf Agriculture ou Cropin: Para integração de dados de fazenda/IoT (mais enterprise).

Considerações para SaaS
Prioridade Brasil → Comece com Embrapa AgroAPI (AGROFIT + Agritec/Soil) + SoilGrids. Isso cobre ~70-80% do que você pediu com dados oficiais/atualizados.
Custos → Muitas são gratuitas/freemium (SoilGrids, OpenWeather free tier, AgroAPI depende de plano). Satélite/clima premium sobe rápido com volume.
Precisão Regional → Soja/milho no Brasil variam muito por microclima (ex: MATOPIBA, Cerrado vs Sul). Combine com ZARC + coordenadas do usuário.
Outras ideias interessantes → Integração de recomendação de adubação (baseada em solo + cultura), alerta de pragas (via clima + defensivos), histórico NDVI para safra anterior, ou integração com máquinas (John Deere API etc.).

2. APIs para Microrregiões e Indicação de Cultivares de Soja e Milho

Agritec API (Embrapa AgroAPI): Freemium (1 mês grátis com 1.000 requests), estima produtividade de soja e milho por microrregião IBGE, com indicações de cultivares adaptadas (ex.: variedades resistentes a seca no Cerrado). Integra com zoning agrícola para datas de plantio. Perfeita para recomendações personalizadas.

Portal de Cultivares (Embrapa): Datasets abertos para cultivares registradas, integráveis via scraping ou AgroAPI. Inclui adaptações por bioma (ex.: soja para Sudeste).

AGROFIT Consulta Aberta (MAPA): Portal gratuito do Ministério da Agricultura, com dados abertos para defensivos. Não API nativa, mas use web services ou datasets CSV para automação. Site: https://agrofit.agricultura.gov.br/.
Bioinsumos API (Embrapa): Complementar, gratuita para produtos biológicos alternativos a químicos.
